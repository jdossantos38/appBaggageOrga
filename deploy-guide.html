📱 Déployer l'application sur smartphone

Installer Liste de Bagages sur iPhone ou Android via GitHub Pages

🌍 Principe

L'application est hébergée gratuitement sur GitHub Pages. C'est un service de GitHub qui publie automatiquement vos fichiers comme un site web. Comme l'app est une PWA (Progressive Web App), elle peut ensuite être installée sur l'écran d'accueil de n'importe quel smartphone et fonctionner hors-ligne.

Partie A — Activer GitHub Pages

1 Se connecter à GitHub

Allez sur https://github.com et connectez-vous à votre compte.

2 Ouvrir le dépôt

Naviguez vers votre dépôt :

https://github.com/jdossantos38/appBaggageOrga
3 Accéder aux paramètres Pages
  1. Cliquez sur l'onglet ⚙️ Settings (en haut du dépôt)
  2. Dans le menu latéral gauche, cliquez sur Pages (sous la section "Code and automation")
4 Configurer la source

Dans la section "Build and deployment" :

  1. Source : sélectionnez Deploy from a branch
  2. Branch : sélectionnez main
  3. Folder : laissez / (root)
  4. Cliquez sur Save
💡 Le déploiement prend généralement 1 à 3 minutes. Vous pouvez suivre l'avancement dans l'onglet Actions du dépôt.
5 Récupérer l'adresse du site

Une fois le déploiement terminé, GitHub affiche l'URL de votre site dans la page Settings → Pages :

https://jdossantos38.github.io/appBaggageOrga/

C'est cette adresse que vous utiliserez sur votre smartphone.

💡 Si l'URL n'apparaît pas immédiatement, patientez quelques minutes et rafraîchissez la page.

Partie B — Installer sur iPhone (iOS / Safari)

1 Ouvrir Safari

Sur votre iPhone ou iPad, ouvrez Safari (le navigateur d'Apple). L'installation PWA ne fonctionne qu'avec Safari sur iOS.

⚠️ Chrome ou Firefox sur iPhone ne permettent pas d'installer des PWA. Utilisez obligatoirement Safari.
2 Accéder à l'application

Tapez l'adresse dans la barre d'URL :

https://jdossantos38.github.io/appBaggageOrga/

Attendez que la page se charge complètement.

3 Ouvrir le menu Partager

Appuyez sur le bouton Partager (icône carrée avec une flèche vers le haut ↑) situé en bas de l'écran Safari.

4 Ajouter à l'écran d'accueil
  1. Faites défiler les options et appuyez sur « Sur l'écran d'accueil »
  2. Vous pouvez modifier le nom si besoin (par défaut : « Bagages »)
  3. Appuyez sur Ajouter

✅ L'icône de l'application apparaît maintenant sur votre écran d'accueil, comme une vraie app !

5 Utiliser l'application

Ouvrez l'app depuis l'écran d'accueil. Elle s'ouvre en plein écran (sans barre Safari) et fonctionne même sans connexion internet grâce au Service Worker.

Partie C — Installer sur Android (Chrome)

1 Ouvrir Chrome

Sur votre téléphone Android, ouvrez Google Chrome.

2 Accéder à l'application

Tapez l'adresse dans la barre d'URL :

https://jdossantos38.github.io/appBaggageOrga/

Attendez que la page se charge complètement.

3 Installer l'application

Méthode 1 — Bannière automatique

Chrome affiche souvent une bannière en bas de l'écran : « Installer l'application ». Appuyez dessus.

Méthode 2 — Menu Chrome

  1. Appuyez sur le menu ⋮ (3 points verticaux, en haut à droite)
  2. Appuyez sur « Installer l'application » ou « Ajouter à l'écran d'accueil »
  3. Confirmez en appuyant sur Installer

✅ L'application apparaît dans votre tiroir d'applications et sur l'écran d'accueil.

4 Utiliser l'application

Lancez l'app depuis l'écran d'accueil. Elle fonctionne en plein écran et hors-ligne.

Partie D — Mettre à jour l'application

1 Mettre à jour le code

Depuis votre PC, après avoir modifié les fichiers :

  1. Ouvrez un terminal dans le dossier du projet
  2. Exécutez :
    git add -A
    git commit -m "Description de la modification"
    git push

GitHub Pages redéploie automatiquement en 1-3 minutes.

2 Mettre à jour sur le smartphone

Le Service Worker gère les mises à jour. Il suffit de :

  1. Ouvrir l'application sur votre smartphone (avec une connexion internet)
  2. Le nouveau contenu se charge en arrière-plan
  3. Fermer et ré-ouvrir l'application pour voir les changements
💡 Si la mise à jour ne s'affiche pas, forcez un rafraîchissement en fermant complètement l'app puis en la ré-ouvrant. Sur iOS, balayez vers le haut depuis le sélecteur d'apps pour la fermer.

🔧 Dépannage

L'option « Sur l'écran d'accueil » n'apparaît pas (iOS)
  • Vérifiez que vous utilisez Safari (pas Chrome ni Firefox)
  • Vérifiez que l'URL est HTTPS (GitHub Pages le fournit automatiquement)
  • Essayez de recharger la page et réessayez
L'option « Installer » n'apparaît pas (Android)
  • Vérifiez que vous utilisez Chrome
  • Vérifiez que l'URL est HTTPS
  • Essayez via le menu ⋮ → « Ajouter à l'écran d'accueil »
  • Attendez quelques secondes sur la page — la bannière peut mettre du temps à apparaître
La page affiche une erreur 404
  • Vérifiez que GitHub Pages est activé dans Settings → Pages
  • Vérifiez que la branche est bien main et le dossier / (root)
  • Attendez 2-3 minutes après l'activation — le déploiement prend du temps
  • Consultez l'onglet Actions du dépôt pour voir si le build a réussi
L'app ne se met pas à jour
  • Vérifiez que le numéro de cache dans sw.js a été incrémenté (ex: bagages-v5 → bagages-v6)
  • Ouvrez l'app avec une connexion internet, fermez-la complètement, puis ré-ouvrez
  • En dernier recours : supprimez l'app de l'écran d'accueil et réinstallez-la

📱 Deploy the App on Smartphone

Install Trip Packing List on iPhone or Android via GitHub Pages

🌍 Overview

The app is hosted for free on GitHub Pages, a GitHub service that automatically publishes your files as a website. Since the app is a PWA (Progressive Web App), it can be installed on any smartphone's home screen and works offline.

Part A — Enable GitHub Pages

1 Log in to GitHub

Go to https://github.com and sign in to your account.

2 Open the Repository

Navigate to your repository:

https://github.com/jdossantos38/appBaggageOrga
3 Open Pages Settings
  1. Click the ⚙️ Settings tab (at the top of the repo)
  2. In the left sidebar, click Pages (under "Code and automation")
4 Configure the Source

In the "Build and deployment" section:

  1. Source: select Deploy from a branch
  2. Branch: select main
  3. Folder: keep / (root)
  4. Click Save
💡 Deployment usually takes 1-3 minutes. You can track progress in the repo's Actions tab.
5 Get the Site URL

Once deployed, GitHub shows your site URL on the Settings → Pages page:

https://jdossantos38.github.io/appBaggageOrga/

This is the address you'll use on your smartphone.

💡 If the URL doesn't appear right away, wait a few minutes and refresh the page.

Part B — Install on iPhone (iOS / Safari)

1 Open Safari

On your iPhone or iPad, open Safari (Apple's browser). PWA installation only works with Safari on iOS.

⚠️ Chrome or Firefox on iPhone cannot install PWAs. You must use Safari.
2 Navigate to the App

Type the address in the URL bar:

https://jdossantos38.github.io/appBaggageOrga/

Wait for the page to fully load.

3 Open the Share Menu

Tap the Share button (square icon with an upward arrow ↑) at the bottom of the Safari screen.

4 Add to Home Screen
  1. Scroll through the options and tap "Add to Home Screen"
  2. You can edit the name if needed (default: "Bagages")
  3. Tap Add

✅ The app icon now appears on your home screen, just like a native app!

5 Use the App

Open the app from your home screen. It opens in full screen (no Safari bar) and works even without internet thanks to the Service Worker.

Part C — Install on Android (Chrome)

1 Open Chrome

On your Android phone, open Google Chrome.

2 Navigate to the App

Type the address in the URL bar:

https://jdossantos38.github.io/appBaggageOrga/

Wait for the page to fully load.

3 Install the App

Method 1 — Automatic Banner

Chrome often shows a banner at the bottom: "Install app". Tap it.

Method 2 — Chrome Menu

  1. Tap the ⋮ menu (3 vertical dots, top right)
  2. Tap "Install app" or "Add to Home screen"
  3. Confirm by tapping Install

✅ The app appears in your app drawer and on your home screen.

4 Use the App

Launch the app from your home screen. It runs full-screen and works offline.

Part D — Update the App

1 Push Code Changes

From your PC, after editing files:

  1. Open a terminal in the project folder
  2. Run:
    git add -A
    git commit -m "Description of change"
    git push

GitHub Pages redeploys automatically in 1-3 minutes.

2 Update on Smartphone

The Service Worker handles updates. Simply:

  1. Open the app on your smartphone (with internet connection)
  2. New content loads in the background
  3. Close and reopen the app to see changes
💡 If the update doesn't show, force-close the app and reopen it. On iOS, swipe up from the app switcher to fully close it.

🔧 Troubleshooting

"Add to Home Screen" doesn't appear (iOS)
  • Make sure you're using Safari (not Chrome or Firefox)
  • Make sure the URL is HTTPS (GitHub Pages provides this automatically)
  • Try reloading the page and try again
"Install" doesn't appear (Android)
  • Make sure you're using Chrome
  • Make sure the URL is HTTPS
  • Try the ⋮ menu → "Add to Home screen"
  • Wait a few seconds on the page — the banner may take time to appear
Page shows 404 error
  • Make sure GitHub Pages is enabled in Settings → Pages
  • Make sure the branch is main and folder is / (root)
  • Wait 2-3 minutes after enabling — deployment takes time
  • Check the Actions tab in the repo to see if the build succeeded
App doesn't update
  • Make sure the cache number in sw.js was incremented (e.g. bagages-v5 → bagages-v6)
  • Open the app with internet, fully close it, then reopen
  • Last resort: remove the app from home screen and reinstall it
